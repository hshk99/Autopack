## Part 2: Customer Segment Analysis (ALL Segments)

### Segment 1: General Users (Largest Pool)

**Size**: ~200M global desktop users organizing personal files
**Characteristics**:
- Consumers, students, freelancers
- Organize photos, documents, downloads, personal archives
- Current solutions: Manual folders, Sparkle (Mac), built-in OS features

**Pain Points**:
1. **Manual organization is tedious** (hours spent organizing)
2. **Can't find files later** (poor search, inconsistent naming)
3. **Duplicate files everywhere** (wasted storage)
4. **No context understanding** (OS doesn't know "tax docs 2023" from "tax docs 2024")
5. **Platform lock-in** (Sparkle is Mac-only)

**Willingness to Pay**: $5-$15/month or $50-$100 one-time
**Customer Acquisition Cost (CAC)**: $30-$50 (content marketing, SEO, word of mouth)
**Lifetime Value (LTV)**: $80-$150 (avg $10/mo × 8-15 months retention OR $80 one-time)
**LTV/CAC Ratio**: 1.6-3.0 (⚠️ MARGINAL - needs optimization)

**Why They'd Switch FROM Sparkle**:
- Sparkle is Mac-only (we're cross-platform)
- Sparkle has limited control (we offer granular rules)
- Sparkle is imprecise categorization ([source](https://reneedefour.medium.com/21-days-with-sparkle-a-review-of-everys-ai-file-organizer-for-mac-c543b51d027d))
- Sparkle is one-time cleanup, not long-term ([source](https://matthewbecher.substack.com/p/mini-review-sparkle-file-organizer))

**Realistic Switching Likelihood**: **MEDIUM** (30-40% if we're significantly better)

### Segment 2: Legal Professionals (Highest WTP)

**Size**: ~5M lawyers/paralegals globally (US: 1.3M lawyers, global est.)
**Characteristics**:
- Solo practitioners, small law firms (1-10 attorneys)
- Organize case files, evidence, discovery, pleadings
- Current solutions: CaseMap+ ($$$), ChronoVault ($$$), Casefleet ($$$), manual folders

**Pain Points**:

3. **Steep learning curve** (enterprise UIs are complex)
4. **No timeline for individuals** (timeline tools are enterprise-only)
5. **Manual evidence organization** (hours per case)

**Willingness to Pay**: $50-$150/month or $300-$500/year
**Customer Acquisition Cost (CAC)**: $200-$400 (legal blogs, bar associations, LinkedIn ads)
**Lifetime Value (LTV)**: $1,200-$3,600 (avg $100/mo × 12-36 months OR $400/yr × 3-9 years)
**LTV/CAC Ratio**: 3.0-9.0 (✅ EXCELLENT - highest priority segment)

**Why They'd Switch FROM ChronoVault/Casefleet**:
- Enterprise pricing ($100-$500/mo vs our $50-$100/mo)
- Cloud-only (we offer local-first)
- Overkill features (we focus on solo practitioners)
- Desktop app vs web-only

**Realistic Switching Likelihood**: **MEDIUM-HIGH** (40-60% if pricing + privacy compelling)

## Part 4: Switching Cost Analysis

### Why Would Users Switch FROM Competitors TO Us?

#### From Sparkle (Mac General Users)

**Their Strengths**:
- Mac-native, beautiful UI
- AI-powered automatic organization
- One-time purchase ($5) or cheap subscription
- Personalized folder structure
- Works well for initial cleanup

**Their Weaknesses** ([sources](https://reneedefour.medium.com/21-days-with-sparkle-a-review-of-everys-ai-file-organizer-for-mac-c543b51d027d), [review](https://matthewbecher.substack.com/p/mini-review-sparkle-file-organizer)):
- **Mac-only** (excludes Windows/Linux users - 70% of market)
- **Limited control** (can't customize AI sorting rules granularly)
- **Imprecise categorization** (relies on file names, not content)
- **One-time cleanup tool** (not long-term file management)
- **No advanced features** (no OCR, no timeline, no legal features)

**User Pain Points/Complaints**:
- "I don't think I need Sparkle long-term" (not sticky)
- "Imprecise sorting" (AI doesn't understand content)
- "Limited control over AI's sorting rules" (power users frustrated)
- "Manual adjustments needed" (defeats purpose of automation)

**Switching Barrier**: **LOW-MEDIUM**
- Time: <1 hour (re-organize files with new tool)
- Money: $5-$10 sunk cost (minimal)
- Learning curve: Low (similar AI-powered approach)
- **Total barrier**: <$100 equivalent

**What We Must Offer to Win Sparkle Users**:
1. **Cross-platform** (Win/Mac/Linux) - captures 70% they can't
2. **Content understanding** (OCR + LLM, not just filenames) - 10x better categorization
3. **Granular control** (let users define rules + AI suggestions) - power user friendly
4. **Long-term value** (continuous organization, not one-time) - sticky product
5. **Advanced features** (timeline, cross-references, search) - differentiation

**Is It Enough?** **YES** - if execution is excellent
- Cross-platform alone captures 70% of market Sparkle can't
- Content understanding addresses #1 complaint (imprecise categorization)
- Granular control addresses #2 complaint (limited control)

**Realistic Switching Likelihood**: **40-50%** (if we're significantly better on content understanding + cross-platform)

## Part 8: Technical Feasibility & Risk

### Technology Limitations

**OCR Accuracy**:
- **Tesseract**: 30-70% accuracy on complex docs (handwriting, low-quality scans)
- **GPT-4 Vision**: 80-90% accuracy but $0.01-$0.03 per page (cost risk at scale)
- **Risk**: If OCR disappoints, legal users churn (evidence must be accurate)
- **Mitigation**: Hybrid approach (Tesseract default, GPT-4 Vision fallback), confidence scores

**Local LLM Performance**:
- **SaulLM-7B / Qwen2-7B**: Can handle legal context BUT requires 16GB RAM (excludes older computers)
- **Risk**: Users with 8GB RAM can't run local LLM (forces cloud LLM, privacy concerns)
- **Mitigation**: Cloud LLM opt-in for low-RAM users, warn about privacy trade-off

**Cross-Platform Consistency**:
- **Tauri WebView differences**: Edge (Windows), WebKit (Mac), WebKitGTK (Linux)
- **Risk**: UI bugs, rendering differences, inconsistent behavior
- **Mitigation**: Extensive testing on all 3 platforms, QA process

**Legal Feature Complexity**:
- **Timeline generation**: Requires accurate date extraction (OCR + NLP)
- **Cross-reference tracking**: Requires dependency graph (complex data structure)
- **Risk**: Features don't work well enough for legal professionals (churn)
- **Mitigation**: Beta testing with real lawyers, iterate based on feedback
